Machine Learning

Machine Learning is Artificial Intelligence are two things that are spoken of as if they are separate from one another. The reality is that machine learning is a more complex branch of your typical artificial intelligence. That’s not to say that they aren’t vastly different in the way that these algorithms/programs are created. Simply put, artificial intelligence is a program that is written to carry out a simple task as a mimicry of human behaviour, a very basic example of this would be using a program to sort emails that contain the word ‘sale’ into a junk folder (or different subfolder) in your inbox. Machine learning however, is closer to being a mimicry of the human brain. Machine learning is a program or algorithm that will continually grow and ‘learn’ as time goes on using large datasets instead of hard coded rules. These databases continually grow as time goes by and as a result, end-users of certain platforms are feeding these algorithms more information that is being used to train the algorithm further. The benefit of these kinds of programs is that they are able to carry out tasks without being explicitly programmed to do so.

If you were wondering how machine learning is used today, take a look at your YouTube recommendations, or the “Up Next” feature that has recently been implemented in addition to the autoplay feature. This is how these technologies are used by big companies today to keep us on their websites. When creating the Up Next feature, YouTube realised that this had to be significantly more complex that the recommendation systems of applications such as Spotify. Because YouTube is constantly growing and thousands of videos are uploaded to the website every minute, they needed a system where the up next section would recommend a video based on multiple things.


The biggest features taken into consideration when recommending you a video are:

  • User watch history (topics) and

  • User’s with similar watch histories rankings of an up next ‘candidate’

This feature ties in well with how the recommended section of YouTube works as well. In a similar way, the recommended section of your YouTube page takes into account these same features.

But there’s more to it than just that. It’s no secret that YouTube still wants to make you spend as much time on their platform as they possibly can. To accomplish this, the recommendation algorithm will generate a shortlist of over one-hundred video candidates based on relevant features such as the video topic, watch time, and various other features of the video that the user is currently streaming. The algorithm will then sort those candidates using the user’s likes, clicks, watch times, and other information that it has been fed.

Moving away from YouTubes recommendations and diving deeper into how machine learning algorithms are created. The Two biggest kinds of learning that can be utilized by machine learning systems are known as supervised, and unsupervised learning. Let’s break down exactly what those terms mean.

Firstly, Supervised Learning is when an algorithm is fed an input and an expected output, this is what is known as the training data. Once the machine has been fed enough training data it is then able to process inputs and come up with an estimated output. If the supervisor sees that the machine got its output wrong, then the machine will adjust it’s calculations until it no longer makes any mistakes. An example of unsupervised learning could be something like a weather prediction AI.


Image Credit: ResearchGate

Then we have Unsupervised Learning, which is machine learning using datasets without a specified structure. Essentially the machine is just left to try and make logical classification from data that it is given. Unsupervised learning is particularly useful when it comes to pattern recognition that humans typically do not pick up on. These algorithms are able to discover hidden patterns and analyze on their own. Making them ideal for customer segmentation, exploratory data analysis and more.

To take this a step deeper. The most popular method of machine learning used in the modern world is Deep Learning using Neural Networks. An example of unsupervised learning. This is the kind of machine learning (to what we have today) that most closely resembles a living brain. This is because a neural network uses the concept of neurons - which take an input, process that input and produce an output.


Image Credit: FreeCodeCamp

The example used here is for airlines, but the concept is the same across many other topics. Over on the left side we have the input layer. This is where the algorithm will take inputs that are to be processed. The blue circles in the middle are what we call ‘hidden layers’. It’s inside these layers where the machine learning algorithm will process the inputs (or features) that were previously given to produce an output shown by the orange circle labelled ‘price’.

It’s worth noting that the amount of hidden layers in a neural network have a range of two or more. One of the bigger challenges of deep learning networks is deciding how many hidden layers the algorithm will need to be most efficient. Some algorithms obviously need more than others, it depends on the amount of features/columns/examples/etc. It’s within these layers that the information is processed and the machine performs mathematical computations on the inputs provided.

Each connection between the neurons is associated with a weight, which is generally chosen at random to begin with. You can think of the weight as a way of determining the importance of an input value. There are more numbers that go into this deep learning method such as a bias (typically (-1, 1) or {0, 1)). The important thing to remember here is that these computations will use these weights to determine what’s most important and to standardise the outputs given by previous neurons. These computations will occur over and over, becoming more and more accurate with each iteration.


Image Credit: Jalammar

What machine learning means for the future is good things. For example, today machine learning can be used to give a diagnosis for a range of medical issues including breast cancer. While not 100% accurate, the decision boundaries in some algorithms are very close to being able to give proper diagnosis. Given that there are so many rows of medical information located inside databases, it’s no big surprise that machine learning algorithms can process symptoms and doctors diagnosis’ to accurately give machine diagnosis to patients.

As will likely be explained in the Autonomous Vehicles section, machine learning used in these vehicles can take information from a number of cameras and sensors attached to the vehicle and use this information to get a passenger from point A to point B.

Cloud, Services, Servers

Cloud computing can be defined as the delivery of computing services over the internet. The cloud is essentially a data warehouse full of servers used for the purpose of hosting data storage, web applications and more. You can think of data warehouses as large buildings with a lot of air conditioning to support the housing of multiple computer and server racks.

The National Institute of Standards and Technology (NIST), which is responsible for developing standards and guidelines on cloud computing, defines it as a “model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g. networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction.” This cloud model is composed of five essential characteristics, three models and four deployment models (Peter Mell & Timothy Grance, NIST, Page 2)


[the five essential characteristics of cloud computing]
Image Credit: TechCroma

There are five essential characteristics of cloud computing today and those are as follows:

  1. On Demand Self Service - The ability to perform computing processes without requiring interaction with each service provider.

  2. Ubiquitous Network Access - The cloud can be accessed using a multitude of different devices, such as PCs, smartphones, tablets, etc.

  3. Location Transparent Resource Pooling - The clouds computing resources are pooled together to allow for greater efficiency. In other words, clients that use minimal computing power will require less system resources than clients who use large amounts of resources.

  4. Rapid Elasticity - The cloud can be easily scaled as the number of connected clients increase. Clouds typically expand, which is why it is so important for the system to allow for expansion and scalability

  5. Measured Service - The ability to measure the clouds performance in some way, shape or form. (Bandwidth, processing, etc.)

Cloud computing deployment models can be used to identify how servers are utilized and provisioned over the internet. Currently, there are four deployment models that are used today, those being community, private, public, and hybrid.





Community

A community cloud environment is a deployment model which is provisioned for exclusive use by a specific community of consumers. Examples include universities cooperating in certain research fields, or police departments within a country or state sharing computing resources. Access to a community cloud environment is typically restricted to the members of said community.

Private

A private deployment model comes with the name ‘private’. These models are specific to individuals or enterprises, allowing them to deploy their own infrastructure and applications to be maintained and operated on their own secure server.

Public

A public cloud environment is a model offered by a third-party provider. For example, Amazon’s Elastic Compute Cloud, or Microsoft’s Azure. These services are available for public use, where data is hosted on public servers.

Hybrid

A hybrid environment combines the features of both private and public models. This model allows companies and individuals to take advantage of public resources as mentioned above, Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS). All while keeping critical/sensitive data in a traditional private server





As we just mentioned, there are three different types of cloud service models available today. Those being Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS).



Image Credit: Red Hat


IaaS - Infrastructure as a Service

IaaS typically provides access to network, storage and compute resources on a pay as you go basis. Users of this service can access a high level of flexibility and management control over IT resources. Users still manage their own applications, data, runtime and middleware on-premise. But the virtualisation, servers, storage and networking side of things is left to vendors off site.


PaaS - Platform as a Service

Platform as a service is mainly used by developers. It removes the need to manage any underlying infrastructure which is typically the hardware and operating system. This allows the user to focus on the development and management of their applications without the additional concerns of patching and managing their own Operating Systems, runtimes, servers, etc.


SaaS - Software as a Service

SaaS can typically be demonstrated by website applications. These applications and everything needed to run them is hosted and managed externally. This provides a complete product that is run and managed by a service provider. An example of a SaaS product would be the google suite (gmail, docs, slides, sheets, etc.)

With SaaS you don’t have to worry about how the service is going to be kept up to date or maintained as it is all managed off site. You also hold no responsibility when it comes to making sure that these applications aren’t vulnerable to outside attacks - of course that doesn’t mean you can just go giving out your passwords to strangers on the internet.

Cloud computing is becoming such a popular way of running business these days because it eliminates the need to engage with the capital expenses up front, such as purchasing hardware, set-up and running costs of on-premise servers and wages of in-house IT professionals.

This technology has revolutionised the traditional way that individuals and businesses think about data storage, IT resources and the workplace model. Although the concept of having data on remote computers is not new and has been around since the 1960s, the flexibility of services has increased so much due to cloud technologies. The cloud facilitates many positive impacts from unlimited scalability, flexibility, mobility, reliability, remote collaboration, communication and reduced overhead.

As a result of the many benefits of cloud computing, organisations are adopting a cloud-first approach to replacing and implementing systems. As a consequence of this new transition, the IT job market is evolving and the dynamics of the work force in terms of required skills and job positions is continuing to adapt. Jobs with low experience and skill requirements have been affected significantly as a result of the adoption of cloud services. For example, administration, data entry, and secretarial jobs in the market have dramatically decreased in the last decade. Businesses no longer have as much of a need to employ low skilled individuals when they’re able to outsource tasks to autonomous cloud-based services. In terms of changes for IT, it’s becoming apparent that the need for companies to have on-premise system administrators is decreasing and these IT Professionals are being sourced into bigger corporations in order to accommodate for a majority of companies making their move to cloud-based services.

Current job positions such as software engineer, software architect, and DevOps Engineer are becoming more and more popular as the need for these skills is growing with the adoption of the cloud.


Image Credit: seek.com.au

In the foreseeable future, the increasing demand for highly skilled employees will affect other areas of life. Universities will have to adapt the way they are teaching IT Courses as the IT field is continuing to grow. It’s entirely possible that 10 years from now a Bachelor of IT will lose it’s meaning as it won’t indicate any particular skill.




Autonomous Vehicles

Autonomous vehicles (or driverless) cars, are vehicles that use sensors, camera, radars and artificial intelligence to function without human operation. Autonomous vehicles are created to respond to conditions that a human would be able to manage, but their capabilities are even greater as the sensors are always on and the ability to avoid collision from blind spots is far greater. There are many companies experimenting with driverless vehicles of their own such as BMW, Ford, Audi, Volvo and the ever so popular Tesla.

According to the Society of Automotive Engineers (SAE), there are 6 different levels of driving automation as shown here.


Image Credit: SAE


Self-Driving vehicles are being engineered today to solve all kinds of problems caused by human error such as traffic delays and collisions. A driverless vehicle is able to sense and avoid danger much more consistently than an average human, as a result of this, collisions, injury and even death can be prevented in many cases due to artificial intelligence used in automated vehicles.

This technology is still working its way into our everyday lives. Most companies are still developing and trialling their own version of a self-driving car, however, not all have been successful. Those companies which have taken major strides include Waymo, the ‘sister’ company of Google. Waymo’s vehicle managed to accomplish 20 million miles driven only with automated technologies. Compare this to another successful example, General Motors Company, managed to complete almost half a million miles. While not quite as many as 20 million this is still a huge feat.

Tesla also has their own autonomous vehicle, however this technology is only a level 2 vehicle, meaning that the passenger can take control at any given moment. However, even with these advancements, the world has not been able to implement these vehicles into everyday life. Back in 2015, The Guardian predicted that by 2020, everyone will be a permanent back seat driver (Piper 2020). Elon Musk forecasted that by 2018, there will be self-driving cars but when that failed, he said 2020. Now in 2021, those cars are still yet to arrive. Despite the massive efforts put forward by car manufacturers, fully automated vehicles still aren’t filling our streets today as they are only available in trial programs.

Ford expected to have their first level 4 vehicle hit the streets in 2021. However, due to the COVID-19 pandemic the company was forced to rethink its marketing strategy.


Image Credit: Tesla

The Victorian Transport Policy Institute believes that level 5, fully autonomous vehicles will be commercially available by the year 2030. Initially, these cars will be very expensive and would have limited performance. According to the institute’s research, by the year 2045, 40% of transportation will be autonomous. For most companies, self-driving cars are the way of the future, but there is no way for them to know how far into the future this will be for certain.

While these ideas for driverless cars are fascinating and innovative. They have proven to be a lot more difficult than any of these companies could have predicted. Steve Wozniak, co-founder of Apple, believes that the accomplishment of self-driving cars is still a long way off. John Krafcik, CEO of Waymo, believes that driverless cars will never be able to function in all conditions. Despite all ambitions to make driverless cars a part of our lives, many companies are coming to the realisation that these cars are very difficult to produce.

Driverless cars will need extremely intricate systems that utilise a variety of technologies. To dive in deeper to this, there are two key sensors needed for a car to successfully identify its environment, a vision sensor, and a Light Detection and Ranging (LiDAR) Sensor. A LiDAR determines ranges by targeting an object with a laser, and measuring the time it takes for the light to come back. In short, these sensors are used to prevent collisions.


Image Credit: LandMark Dividend



Other technologies utilised by self-driving vehicles includes Advanced Driver-Assistance Systems (ADAS), which covers cruise control, collision warnings, traction control, etc. Another main component in the development of self-driving vehicles is AI and deep learning with neural networks (See Machine Learning section), these technologies are used to imitate the human brain in some form. It is said that AI is the lead for launching Level 5 autonomous vehicles, which require zero human interaction.

Unless safety and security are ensured, people aren’t going to feel too comfortable owning and using a self-driving car, which is why the Responsibility-Sensitive Safety (RSS) framework (developed by Intel) is an important component in the making of such vehicles. Another system incorporated by many companies is the adaptive cruise control system. This system ensures that cars maintain a safe distance between other vehicles and will adjust the driving speed as necessary.

In today’s day and age, one of the most common technologies that are being used by engineers are Arduinos as they are a cheaper option than Raspberry Pi’s. But a lot of engineers prefer the Raspberry Pi, Orange Pi and others because they are ARM based computers capable of running multiple programs and multiple pieces of equipment simultaneously. Arduinos on the other hand are simply a programmable board with a microprocessor capable of running a single program.

Both of these options are a low-cost, low-power computer, initially created with the purpose of teaching younger people computer science skills. However, upon release of this technology tinkerer’s everywhere wanted to get their hands on them to create their own innovative projects without breaking the bank. The easiest way to think of a Raspberry Pi is to think of your smartphone, just without the screen, sensors, microphone, speakers, battery, camera, etc.

Raspberry Pi’s have many other practical uses as well. Such as hacking or penetration testing. Many black-hat hackers prefer this technology due to its flexibility, capabilities and compact size, making it easy to hide.

To explore the many capabilities Raspberry Pi’s have to offer, provided below are some explanations of the key components used to create innovative gadgets.



General Purpose Input Output (GPIO)

One of the most powerful features of Raspberry Pi’s today is the strip of GPIO pins located at the top of the board. This GPIO strip is used for connecting the Pi to the outside world. For example, these pins allow you to do a whole variety of things such as turning on/off LEDs to running traffic lights and controlling motors.


Image Credit: Raspberry Pi


GPIO

3.3V, Maximum current draw 16mA (milliAmperes)

GPIO 27, 28

Reserved for advanced use, often times used by HAT boards

GND

Ground pins

GPIO 1

Static Pin, used for supplying extra power

GPIO 2, 4

5V, Max current draw 1A/2A

These two pins are directly connected to the Raspberry Pi’s power supply. With a 3A you can expect to draw minimum 1A current.




Breadboard

A breadboard is especially useful when it comes to prototyping, as it allows you to set up and test your circuits without needing to solder your circuits together.


Image Credit: Tweaking4All

The middle 2 layers (Labelled B and C) are used for creating circuits with lights, motors, resistors, etc. On the outside layers (Labelled A and D) we have the power rail and ground rail. The blue rail is negative, and the red rail is positive.


Resistors

Resistors are a crucial part of all electronics. They are used for many reasons such as to reduce current and divide voltages in a parallel circuit. For example, a 5mm blue LED has values of 3.4V and 30mA. Since both of these values are greater than the GPIO pins max supply (3.3v and 16mA), connecting it directly to the strip will try to pull more than what’s available - damaging the GPIO strip over time. Using Ohm’s Law to solve this problem: V(volts) = I(Amps) x R(Resistance)

3.4V / (.03A - .016A) = 242.86 We will need at least a 250(ohm) resistor.

Useful Tricks

Increasing Resistance

Resistors placed in a series will add to the total resistance of the circuit. In the image below, there are 3 220resistors placed in series, which increases the total resistance of the circuit to 660 (R 1+ R2+ ...RN= RTotal)

Decreasing Resistance

By placing resistance in parallel you’re able to create smaller amounts of resistance across the board. This can be demonstrated by the equation
1/RTotal= 1/R1 + 1/R2 + ... 1/RN

Image Credit: Electronics Tutorials

As you can see here there are three resistances all of 200 or greater. Using the equation above we are able to produce a total resistance of 85.67. In reality, the resistance hasn't decreased, it’s just that the resistors have been made less efficient as a result of increased spread of electricity supply in the circuit.

There are many other electrical components that can be used in circuits to create your own machines. Below is a list and brief description of them


Component

Description

Transistor

A transistor is a semiconductor device that can act like a switch. They have 3 pins. Base, emitter and collector. If there’s a little electricity on base the circuit is closed and electricity can flow. Using a Darlington pair method, the transistor can be used to amplify power supply too.

LDR Resistor (Photoresistor)

This is another type of resistor that is responsive to light. When light shines on the resistor, the material absorbs the light and increases the conductivity of the material - starting the equipment. In the absence of light these resistors are able to reduce power supply to a level that disables the equipment's ability to function. To fully shut down a power supply we can use a transistor

Capacitor

A capacitor charges and discharges electricity into a circuit when necessary. They are useful for blocking DC current (which can be hazardous) and allowing AC flow (which is safer)

Jumper Cable

Can carry electricity from anode to cathode. For example, GPIO to breadboard, breadboard to GPIO, breadboard circuit to another circuit, or breadboard to to external equipment

It emits ultrasonic waves that can’t be heard by humans. Based on the time it takes for the sound to be reflected back, they’re able to calculate distance. This is useful in machines such as the Roomba which can detect obstacles.

A LiDAR uses a laser to identify its surroundings and create a map. These components are currently being used in the iPhone 12 Pro and iPhone 12 Pro Max

OpenCV is a library that makes use of machine learning technology that can be used to identify different objects. It’s predicted that robots in the future will make use of this library.

Project Idea

For our group project, TESS decided to go with the PARKD Motorcycle parking application. Although the project was initially conceived by Emmet, all members of the group decided to pursue the same idea and equally contributed towards the project. On the 30th of June, we came together on MS Teams. We plan to utilize Sam for the back end, Taylor and Emmet for the front end, and Saajid for research and formatting.

Overview

ParkD is a free motorcycle parking app for Sydney. It is an application for people who ride motorcycles daily, whether for work, leisure, or just getting from one place to another. The concept of the app is to create a platform which acts like an Airbnb for bike parking. ParkD is community- rated, which means that people in the community can input a location on the app, for motorbike parking which they know is safe and secure. The app locates the closest community rated parking near a chosen location. ParkD will provide timely motorcycle parking solutions to address the issue of reliable parking in Sydney.

Motivation

There is a major problem in a lot of Australian cities right now when it comes to motorbike parking, with greater Sydney having 1145 thefts in the last twelve months, as shown by NMVTRC. As a result of this, we wanted to create an application that motorcyclists could easily use to find, rent, and occupy parking spaces for two-wheeled vehicles. The idea behind this application is to fill a huge gap in the market by providing riders with peace of mind, knowing that they can find, locate, and reserve safer parking using an app where information is provided by other motorcyclists in the area. It is especially concerning that more than half of the victims of motorbike theft never see their bikes again.

Australian Motorcycle Theft Statistics (blackknighttracking.com)




Description

The beauty of the app is that users can register as parking landlords and rent their secure parking spots to other two wheeled users who require temporary parking solutions. The concept is a community-rated platform which acts like an Airbnb for two wheeled commuters. The app locates the closest community rated parking near a chosen location using geolocation.

To be able to provide our bikers with parking information we are going to rely on input from other users. For example, if someone were to park their motorbike in an available bay and noticed that there were multiple other spots available, they could put the location on a map for other bikers to see. Once another biker arrives at the location with the available parking, they can remove that parking spot from the map. Even if this second biker did not occupy the park themselves, they would still be able to remove it from the map.

There are major aspects of this application that need to be considered, including:

  • Common UI features such as a menu button containing settings and the user's profile setup

  • Interactive Map – An interactive map to help users find their current location and look for available parks in that area.

  • Lease my park - The ability for landlord profiles to add and remove a private parking from an Interactive Map View / Geolocation

  • Technical Support – via Notifications via Messaging with regards to Scheduling bookings, Payment Integration and Ratings/Reviews

  • Ratings/Reviews - We would provide users the option to ‘rate their stay’ if they rented a park of another user.

When developing ideas for this application, our group ran into its fair share of problems. However, the biggest question we had to ask ourselves was how are we going to ensure that when a rider vacates a spot, they mark it as ‘free’ again?

As you will find in the sections below, our solution was to allow the user to input an estimated time that they would leave by. When the time the user entered was approaching (say 15 minutes prior), the application would send a notification to their phone, reminding them that time is almost up AND giving them the option to extend their time.

For all these functions to operate together in a cohesive way, much thought had to be given towards the backend.

QR Code

For every ‘landlord’, we will generate a QR code with an alphanumeric combination and store it in the qr_table DB. The QR code format is Date/time + Username + DOB.


Router/Cellular

If the user inputs something that needs to be routed to the IIS or the SQL server. The packets need to be routed to a public/private DNS first and resolve the name to IP address, then the request will be routed to the IP address (IIS, SQL server).

To use a public DNS server(eg. 8.8.8.8) we will need a static public IP address and port forwarding on our router on the web server end. Otherwise, we can also use DynDNS or No-IP. This basically keeps updating the IP address of a server in the public DNS, so even if the IP is dynamic, it can still be resolved from a Domain name.


Web Server/IIS

The website will be hosted by windows server 2016 or 2012 r2. It can also be hosted by Windows 10 if the computer has IIS service installed in it. For cloud hosting Wix, Azure, AWS, GCP, IBM Cloud can be used.


SSL/TLS

For the prototype we may not use an SSL certificate, however in real life it is crucial to have an SSL. http:// does not encrypt any communication between the user and server. So, every communication is plain text and vulnerable.


DNS (Private/Public)

To put it simply, DNS is a translator, it translates a domain name into an IP address. It can be private or public. Private DNS servers are mostly used for intranets. It can also be used for WAN. We will try both DNS and No-IP. No-IP is a dynamic DNS, which keeps a record of updated IP addresses. So, the domain name resolves to the updated IP address.


SQL Server (RDBMS)

A server that contains the database of QR codes, vehicle information and App essentials. Please note that the user_cred (credentials) is in the DMZ zone. So, no other table has relations with them. We have put the user_cred into the DMZ zone for security concerns, such as: SQL injection.

We will have Landlord DB and Tenant DB on different SQL servers. So, even if one physical machine is down the other mode can still work.
